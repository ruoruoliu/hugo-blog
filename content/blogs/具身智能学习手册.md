## 场景理解

### 分割

#### SAM（Segment Anything Model）

![image.png|300](https://images.ruoruoliu.com/2026/03/877fd2df9600bf93b1605729e3507167.png)

通过多种方式引导分割：
- 点：在物体上点一下，它就能自动识别出整个物体的边界
- 框：拉一个框，它会分割出框内的物体
- 文本：根据描述（如“那只猫”）来寻找物体
- 遮罩（Mask）：利用已有的粗略遮罩进行精细化调整

![image.png](https://images.ruoruoliu.com/2026/03/988d145aab074633c48cd72e83ee5d79.png)

实现原理：
- SAM通过大规模预训练（SA-1B），具有zero-shot的泛化能力
- 模型结构基于transformer架构：
	- Image Encoder：Transformer (ViT)，负责把图片转化为特征向量
	- Prompt Encoder：负责处理用户输入的点、框或文本
	- Mask Decoder：轻量级的解码器，能以毫秒级的速度将图像特征和提示结合，实时输出分割结果

SAM 2实现了视频分割，SAM3D实现了三维场景的分割

参考链接：
- [Segment Anything](https://arxiv.org/pdf/2304.02643)

### 检测

#### Open-Voc Detection

![image.png|500](https://images.ruoruoliu.com/2026/03/cc80605cff2ec7da04635186c7b97777.png)

相比于传统的检测，最后一步是Softmax分类，开放词表检测最后一步是特征对齐：
- 视觉分支：提取图像中潜在物体的特征向量（Region Embeddings）
- 文本分支：利用预训练模型（如 CLIP）的Text Encoder，将你的输入转化为语义向量
- 匹配：计算视觉特征和文本特征的余弦相似度，取相似度最高的区域

### 多模态Grounding

根据给定的自然语言描述，在图像或视频中精准地找到并标记出对应的目标，打通了nlp和cv之间的桥梁，升级版（像素级）的clip（clip只做整体语义匹配）

![image.png|400](https://images.ruoruoliu.com/2026/03/6be348371b45f35f3a5ff494f4881fb3.png)

进一步的，实现3D环境的多模态Grounding：
- 多模态输入：2D视觉、3D点云、系统提示词和用户指令
- 特征融合：将输入转化为token序列
- 大脑中心：基于LoRA微调的LLM，负责决策和推理
- 输出形式：文本响应或者动作响应

![image.png|500](https://images.ruoruoliu.com/2026/03/86ad07bf4daeaa41ccab517a5f02bcae.png)

参考链接：
- [GLaMM: Pixel Grounding Large Multimodal Model](https://arxiv.org/pdf/2311.03356)
- [An Embodied Generalist Agent in 3D World](https://arxiv.org/pdf/2311.12871)

## 数据引导

机器人的数据缺失，通过以下方式补充：
- 通过人类操作的视频，模仿学习，比如[MimicPlay](https://arxiv.org/pdf/2302.12422)和[Vid2Robot](https://arxiv.org/pdf/2403.12943)
- 轻量级硬件数据采集：通过成本低的硬件设备大量采集数据，比如[UMI](https://arxiv.org/pdf/2402.10329)（机械爪）、[DexCap](https://arxiv.org/pdf/2403.07788)（五指手套）、[HIRO Hand](https://arxiv.org/pdf/2309.14860)（人类手把手握着手套，避免DexCap后续适配机器人的问题）
- 重量级硬件数据采集：VR手套+VR眼睛
- 生成式仿真：[RoboGen](https://arxiv.org/pdf/2311.01455)通过基于设定任务生成环境然后进行任务学习；[MimicGen](https://arxiv.org/pdf/2310.17596)基于少量的人类数据样本，分段后分别进行多样化，从而扩充数据量，相对来说更可控

## 动作执行

### 生成式模仿学习

基于神经网络学习概率分布 $P(action|observation)$，从而解决传统的模仿学习（如行为克隆 BC）对于多峰分布（同一个动作序列有多种正确解）的缺陷

#### ACT（Action Chunking with Transformers）

模型基于训练的解码器，针对某一特定任务，预测生成连续动作序列：
- 利用CVAE的思路，基于中间隐状态条件生成
- 训练阶段：
	- 编码器基于当前状态和人类动作序列编码得到Z
	- 解码器通过encoder先把机器人的状态、图像以及Z编码，然后解码成动作序列，目标与人类动作序列一致
- 推理阶段：
	- 让解码器可以基于机器人状态、图像，生成后续动作序列

![image.png](https://images.ruoruoliu.com/2026/03/3451d7228b500d65270fc5f77821ebb2.png)

通过将时序预测结果加权平均，使得到的动作序列更平滑

![image.png|400](https://images.ruoruoliu.com/2026/03/1aec8e699700d81886e165a5157ebbc4.png)

参考链接：
- [Learning Fine-Grained Bimanual Manipulation with Low-Cost Hardware](https://arxiv.org/pdf/2304.13705)

#### Diffusion Policy

![image.png](https://images.ruoruoliu.com/2026/03/cdb603ffc110c993d21a01f640a20c3b.png)

策略不直接输出动作，而是学习分数函数（ $\nabla \log P(A|O)$），通过多次迭代优化得到动作：
- 加噪阶段：在训练时，将人类真实的动作序列逐渐加入高斯噪声，直到变成纯噪声
- 去噪阶段：训练一个神经网络，学习如何根据当前的观察，将噪声一步步剔除，恢复成真实的动作序列

参考链接：
- [Diffusion Policy: Visuomotor Policy Learning via Action Diffusion](https://arxiv.org/pdf/2303.04137)

### Affordance

预测观察中物品的哪些部分是可以接触的，且接触的方式是什么（扭动、提起等）
- 代表有RoboAffordance、AffordPose以及3D场景的SceneFun3D

### Query from LLM

直接基于LLM对物品的接触和使用方式进行询问，得到机器人的动作位置和类型
- 代表有ManipLLM和ManipVQA

### Prompt Planning from LLM

直接基于LLM进行任务的拆解和规划
